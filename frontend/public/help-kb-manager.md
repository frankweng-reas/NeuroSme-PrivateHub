# Knowledge Base 使用說明

## 這個工具是做什麼的？

**Knowledge Base（KB 管理）** 是知識庫的**建置與管理工具**，供知識管理員上傳文件、維護索引、測試查詢效果。

建好的知識庫要讓員工或客戶使用，需要在 **KB Bot Builder** 部署成 Bot，使用者透過 Bot 查詢。

```
Knowledge Base（建庫、上傳、測試）
        ↓  確認效果後設為「發布」
KB Bot Builder（建立 Bot、設定存取控制）
        ↓  分享連結或嵌入網頁
員工 / 客戶（透過 Bot 查詢）
```

---

## 知識庫範圍

建立知識庫時需選擇範圍，決定這個知識庫的用途：

| 範圍 | 說明 | 可部署 Bot |
|------|------|-----------|
| 🔒 **個人私有** | 只有自己看得到，用於草稿與測試 | 否 |
| 🚀 **發布** | 準備好對外服務的知識庫，可供 Bot Builder 引用 | 是 |

> 建立時預設為個人私有。需要 **manager 以上**才能將範圍設為「發布」。

**典型流程**：先建立個人私有知識庫 → 上傳文件 → 右欄測試查詢驗證效果 → 確認後改為「發布」→ 至 Bot Builder 建立 Bot。

---

## 介面說明

介面分為三欄：

| 欄位 | 功能 |
|------|------|
| **左欄** | 知識庫列表，新增/切換/設定/刪除知識庫 |
| **中欄** | 文件管理、查詢統計、資料來源（Connector）、健診 |
| **右欄** | 測試查詢，驗證知識庫回答效果 |

---

## 1. 左欄：知識庫列表

顯示自己建立的所有知識庫。admin 另可看到其他人建立的知識庫。

**發布**的知識庫會顯示 🚀 圖示，以及目前引用它的 Bot 數量。

### 操作

| 操作 | 說明 |
|------|------|
| **＋（新增）** | 建立新知識庫，填入名稱與範圍後點「建立」 |
| **切換** | 點選任一知識庫，中欄與右欄即切換 |
| **⋯ → 設定** | 修改名稱、LLM 模型、知識庫範圍 |
| **⋯ → 轉移所有權** | 將知識庫轉移給其他成員（admin 限定） |
| **⋯ → 刪除** | 刪除知識庫及其所有文件，**無法復原** |

---

## 2. 中欄：文件管理

### 上傳文件

支援格式：**PDF**（`.pdf`）、**Word**（`.docx`）、**純文字 / Markdown**（`.txt`、`.md`）。

上傳時選擇**文件類型**，決定切塊方式與檢索策略：

| 類型 | 適合什麼文件 | 切塊方式 |
|------|------------|---------|
| 📄 **一般文件** | 操作手冊、規章、合約、報告 | 段落切塊（約 1500 字），保留上下文 |
| 💬 **精準問答** | Q&A 問答集、常見問題、客服話術、SOP | 小切塊（約 300 字）＋ BM25 關鍵字索引，**精確直答 Bot 專用** |
| 📑 **整份查閱** | 價目表、術語表、不宜拆分的對照文件 | 整份不切，查詢時一次取回完整內容 |
| 📋 **結構化 MD** | Doc Refiner 產出的 `.md` 文件 | 依 `##`/`###` 標題切 chunk，附章節麵包屑，支援 BM25 |

> **建議**：問答型內容先用 **Doc Refiner（FAQ 模式）** 整理再匯入；多章節手冊用 **Doc Refiner（結構化 MD）** 匯入，效果最佳。

## 3. 中欄：查詢統計

選取知識庫後切換至「查詢統計」標籤，可查看：

- **摘要**：近期查詢總數、命中率、零命中數量
- **熱門問題**：被問最多次的問題清單
- **零命中問題**：查不到答案的問題，用於發現知識庫缺口

---

## 4. 中欄：整合（Connector）

透過「整合」標籤，可將外部平台（目前支援 **Slack**）的內容自動同步至知識庫。

### 新增 Slack 連接器

1. 切換至「資料來源」標籤，點擊「＋ 新增連接器」。
2. 輸入 **Slack User Token**（格式：`xoxp-…`），點擊「驗證並載入頻道」。
3. 勾選要同步的頻道，設定名稱、同步頻率與範圍後，點擊「建立並立即同步」。

> Token 需具備：`channels:history`、`channels:read`、`users:read`、`files:read`。前往 [api.slack.com/apps](https://api.slack.com/apps) 建立應用程式取得。

### 連接器狀態與操作

| 狀態 / 操作 | 說明 |
|------------|------|
| **同步中** | 正在拉取資料 |
| **已啟用** | 依排程自動同步 |
| **暫停** | 自動同步停用，仍可手動觸發 |
| **立即同步** | 增量同步（只拉上次後的新增） |
| **重新同步** | 完整重新拉取所有歷史訊息 |
| **刪除** | 移除連接器（不刪除已同步文件） |

---

## 5. 條目管理（知識條目 Tab）

切換至「知識條目」標籤，顯示知識庫中所有切分後的段落（chunk）。

### 新增條目

點擊「**新增條目**」按鈕，手動輸入 Q&A 或任意文字內容。手動新增的條目統一歸屬在虛擬文件「**手動新增**」之下，方便與上傳文件區分來源。

### 搜尋與篩選

- **搜尋**：在搜尋框輸入關鍵字，即時篩選條目內容。
- **來源篩選**：從下拉選單選擇特定文件，只顯示該文件的條目。

### 條目操作

| 操作 | 說明 |
|------|------|
| **編輯** | 修改條目內容後自動重新 Embedding 與 BM25 索引 |
| **刪除** | 移除不需要的條目 |
| **全選 / 批次刪除** | 勾選多個條目後可一次刪除 |

> 僅知識庫建立者（Owner）與 admin / super_admin 可操作條目管理。

---

## 6. 健診（重複條目掃描）

健診功能可掃描同一知識庫內語意高度相似的條目配對，協助發現並清理重複內容。

### 使用方式

1. 選取知識庫，切換至「**健診**」標籤。
2. 拖動**相似度門檻**滑桿（預設 0.90，值越高代表要求越相似才列出）。
3. 點擊「**開始健診**」，掃描進度即時顯示。
4. 掃描完成後，列出所有疑似重複的條目配對，每組顯示：
   - 相似度百分比
   - 兩個條目的內容與來源文件
5. 對每組配對，選擇**刪除上方**或**刪除下方**，刪除後該配對自動從列表消失。

> 健診使用已存在的 Embedding 向量比對，**不需要額外 API 呼叫**，費用為零。建議在大量匯入內容後執行一次。

---

## 7. 右欄：測試查詢

供知識庫建立者在部署 Bot 前，驗證知識庫的回答效果。

- 選取知識庫，在右欄輸入問題即可測試。
- 對話記錄會保留，點擊右上角重新整理圖示可清除。

---

## 注意事項

- 刪除知識庫將一併刪除其所有文件與向量索引，**無法復原**。
- 上傳大型文件（> 10 MB）可能需要較長時間完成向量化。
- 知識庫設為「發布」後即可被 Bot Builder 引用；建議由管理員統一維護內容品質。
- 知識庫已被 Bot 引用時，修改或刪除文件前會顯示警示，提醒變更將立即影響線上 Bot。
