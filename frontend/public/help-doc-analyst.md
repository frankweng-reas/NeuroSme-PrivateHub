# Doc Analyst 文件分析助理 使用說明

## Overview

**Doc Analyst** 是以文件為中心的 AI 問答助理，上傳 PDF 或 Markdown 文件後，即可針對文件內容進行深度問答、摘要與比較分析，所有對話都與文件綁定，隨時可切換回來繼續提問。

| 典型使用情境 | 說明 |
|-------------|------|
| 快速摘要 | 上傳長篇報告，讓 AI 整理重點 |
| 比較分析 | 上傳多份文件，詢問差異或共同點 |
| 條款解讀 | 上傳合約或法規，詢問特定條文的意義 |
| 技術文件查詢 | 上傳操作手冊，詢問某功能的操作步驟 |
| Doc Parse 延伸 | 在 Document Parse 頁面萃取文字後，可直接跳轉至本 Agent 繼續分析 |

> 若需要跨多份文件的知識庫檢索，請改用 **Knowledge Base**（RAG）；若只是一般對話，請改用 **Chat**。

---

## 快速流程：上傳文件 → 閱讀 AI 摘要 → 提問

---

## 1. 文件管理（左側欄）

### 上傳文件

- 點擊「**上傳新文件**」選擇檔案
- 或將檔案**拖曳至左欄**任何位置

| 支援格式 | 說明 |
|---------|------|
| **PDF**（`.pdf`） | 文字型 PDF（含掃描 PDF 的文字層） |
| **Markdown**（`.md`） | Doc Refiner 或 Doc Parse 產出的結構化文件 |

- 一次可選取**多個檔案**，所有檔案歸入同一個分析對話
- 上傳完成後，AI 會自動讀取文件並產生**摘要 + 建議問題**

### 分析記錄列表

- 左欄列出所有歷史分析記錄，每筆記錄對應一組文件
- 顯示格式：**PDF** 或 **MD** 標籤 + 檔案名稱 + 相對時間
- 點擊記錄即可切換，恢復該份文件的完整對話歷史
- 右上角 **✕** 可刪除記錄（無法復原）

### 折疊 / 展開

- 點擊「**`<`**」折疊左欄，獲得更大的對話空間
- 折疊後點擊 **`>`** 還原

---

## 2. AI 自動摘要與建議問題

上傳文件完成後（或切換至從未對話過的記錄時），系統會自動：

1. 讓 AI 閱讀全文並產生**文件摘要**（自動顯示在對話框，不計入提問次數）
2. 在摘要下方顯示 **3–5 個建議問題**（藍色圓角 Chip 按鈕）

**建議問題使用方式：**
- 直接點擊 Chip 即可送出，不需自己打字
- 送出任一問題後，建議問題列表會消失
- 之後仍可自行輸入問題繼續對話

---

## 3. 文件問答

### 輸入問題

- 在下方輸入框輸入問題，按 **Enter** 或「送出」按鈕發送
- AI 會依文件內容及對話歷史（最近 8 輪）回答

### 模型選擇

- 對話框頂端可切換 **AI 模型**，偏好會自動記憶
- 若尚未設定模型，送出按鈕會顯示提示且無法操作

### 範例問題

- 「請列出本文件的五個重點」
- 「第三章提到的注意事項有哪些？」
- 「請比較這兩份文件在規格上的差異」（需同時上傳兩份文件）
- 「請將文件內容翻譯成英文摘要」

---

## 4. 從 Document Parse 跳轉

在 **Document Parse**（文件解析）頁面完成 OCR 或文字萃取後，可點擊「**在 Doc Analyst 分析**」按鈕：

- 系統自動將萃取的文字傳入 Doc Analyst
- 無需手動下載再重新上傳
- 跳轉後會立即開始 AI 摘要

---

## 注意事項

- **檔案體積**：單次上傳（可多選）加總上限為 **20 MB**（超過會被 Nginx 拒絕）
- **文字字元數**：每個檔案萃取後超過 **60,000 字元**（約 100–150 頁）會被自動截斷；多檔合併後超過 **150,000 字元**（約 250–375 頁）再次截斷。截斷為靜默處理，不會顯示錯誤，建議文件過長時先拆分再上傳
- **不支援圖片格式**：JPG、PNG 等圖片無法直接上傳，上傳後會收到格式錯誤。若要分析圖片內容，請先使用 **OCR Agent** 或 **圖片 → 結構化 MD** 將圖片轉為 `.md`，再上傳至本 Agent
- **掃描 PDF**：若 PDF 為純圖片（無文字層），AI 無法讀取文字，建議先透過 **Document Parse** 或 **Doc Refiner（結構化 MD）** 進行 OCR，再上傳至本 Agent
- **對話歷史**：儲存於伺服器，重新整理或重新登入後仍可查看
- **同一記錄可繼續對話**：切換至其他記錄後再切換回來，對話不會遺失
- **刪除文件記錄**：刪除後，該記錄的所有對話訊息也一併刪除，無法復原
