# Document Parse 使用說明

## 概覽

**Document Parse** 可將 PDF 文件（合約、採購標案、報告等）自動萃取為結構化欄位，大幅縮短人工閱讀時間。

---

## 快速上手

| 步驟 | 操作 |
|------|------|
| **1. 選擇解析類型** | 從左側「解析類型」下拉選擇符合文件類型的 Profile（如「合約解析（通用）」） |
| **2. 上傳 PDF** | 拖曳或點擊上傳區放入 PDF，上限 20 MB |
| **3. 選擇模型** | 選擇要使用的 AI 模型（建議大型文件選 gemma4:26b 或 gpt-4o） |
| **4. 開始解析** | 點擊「開始解析」，系統逐段處理並即時顯示進度 |
| **5. 核對結果** | 對照原文確認各欄位值，可直接點擊修改 |

---

## 解析結果

### 欄位狀態
- **正常顯示**：AI 成功萃取到值
- **紅色標記「未找到」**：文件中找不到此欄位，可點擊手動填入
- **原文依據**：勾選標題列的「原文依據」checkbox，每個欄位下方會顯示 AI 萃取時參考的原文片段（琥珀色區塊），方便核對

### 欄位編輯
- **點擊任意欄位值** → 切換成可編輯狀態
- **Enter** 或 **點擊其他處**：儲存並自動更新資料庫
- **Escape**：取消編輯
- 清單類欄位（如「工作項目」）：每行一個項目

### 匯出
- 點擊標題列的下載圖示（↓）可匯出 **Markdown** 格式報告

---

## 歷史記錄

- 點擊標題列的時鐘圖示（🕐）開啟歷史側邊欄
- 顯示最近 30 筆解析記錄，含檔名、Profile、時間
- **點擊任一筆**：立即還原該次解析結果
- **刪除（垃圾桶）**：從資料庫永久移除該筆記錄
- 回到頁面時自動載入最近一筆結果

---

## Parse Profile（解析設定）

Profile 定義了要從文件中萃取哪些欄位。不同類型的文件應選擇對應的 Profile：

| Profile | 適用文件 |
|---------|---------|
| 標案解析（政府採購） | 政府採購招標文件、投標文件 |
| 合約解析（通用） | 一般商業合約、勞務採購契約 |

### 管理 Profile（限管理員／主管）
點擊左側標題旁的設定圖示（⚙）開啟 Profile 管理員：

- **新增**：從頭建立新 Profile
- **複製**：以現有 Profile 為基礎複製一份再修改
- **編輯**：修改欄位定義、新增／刪除 Section 或欄位
- **刪除**：軟刪除，不影響已有的解析結果

**欄位類型說明：**

| 類型 | 說明 | 範例 |
|------|------|------|
| `text` | 單行文字 | 合約名稱、機關名稱 |
| `text_list` | 多值清單 | 工作項目、風險事項 |
| `currency` | 金額 | 合約金額、押標金 |
| `datetime` | 日期時間 | 簽約日期、投標截止日 |
| `doc_list` | 文件清單 | 應備文件清單 |

---

## 注意事項

- 目前僅支援 **PDF 格式**，純圖片 PDF 需先執行 OCR 才能解析
- 解析結果為 **AI 輔助**，請務必人工核對，特別是金額、日期等關鍵數字
- 「原文依據」功能：若 AI 找不到支持依據，該欄位不會顯示引用區塊（並非錯誤）
- 範本型文件（如空白合約範本）中，留白欄位會顯示「未找到」，此為正常行為

## 常見問題

**Q：為什麼有些欄位全部顯示「未找到」？**
A：若上傳的是空白範本文件（欄位留空），AI 無法從空白中萃取值，屬正常現象。請使用已填寫的實際文件測試。

**Q：不同模型的解析效果有差異嗎？**
A：有。較大的模型（如 gpt-4o、gemma4:26b）通常準確度較高，但速度較慢；gpt-4o-mini 速度快但對複雜條款的理解可能較弱。

**Q：可以解析英文合約嗎？**
A：可以，但目前 Profile 的欄位 hint 以中文撰寫。建議對英文合約建立對應的英文 Profile。
