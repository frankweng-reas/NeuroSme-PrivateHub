# Doc Refiner 使用說明

## Overview

**Doc Refiner** 是知識庫的「前置整理工具」，可將文件或筆記轉成 **結構化 Q&A**，或轉成 **帶章節標題的 Markdown**，再匯入知識庫供 Bot 與搜尋使用。

| 典型使用情境 | 建議模式 | 說明 |
|-------------|---------|------|
| 員工手冊 / HR 規章（問答查詢） | 文件 → FAQ | 萃取 Q&A → 匯入知識庫 → 適合 KB Bot「精準回答」 |
| 產品規格書（重點問答） | 文件 → FAQ | 快速整理規格重點為問答對 |
| 會議記錄 / 草稿筆記 | 筆記 → FAQ | 貼入文字，AI 整合為 Q&A |
| 標準作業程序（步驟不可省略） | SOP → FAQ | 嚴格保留每一步驟 |
| 操作手冊 / 規章 / 含圖表 PDF | 文件 → 結構化 MD | 保留章節結構，低文字頁與內嵌圖自動 OCR |
| 大量 PDF 離線轉換 | 批次 → 結構化 MD | 逐檔轉 MD 並寫入本機資料夾 |

知識管理群組的三個 Agent 分工如下：

- **Doc Refiner**（本 Agent）→ 整理文件 / 筆記後匯入知識庫
- **Knowledge Base** → 管理文件、直接查詢
- **KB Bot Builder** → 以知識庫為基礎部署 Bot

### Q&A 與結構化 MD 怎麼選？

| 匯入類型 | 適合內容 | 知識庫檢索特點 |
|---------|---------|---------------|
| **FAQ（Q&A）** | 問答型、SOP 逐步說明、客服話術 | 小 chunk + 向量 + BM25，適合「問一句答一句」 |
| **structured_md** | 手冊、規章、多章節長文件、含組織圖／表格 | 依 Markdown 標題切 chunk，附章節麵包屑，適合「查某章某節」 |

同一主題可以只選一種；若既有 FAQ 又有完整手冊，也可分別匯入不同知識庫，由 Bot 多選來源。

## 六種模式

透過左側 Sidebar 切換；底部可選 **AI 模型**，各模式整理與改寫皆使用同一模型。

| 模式 | 輸入 | 輸出 | 適用情境 |
|------|------|------|---------|
| **文件 → FAQ** | PDF、TXT（≤ 20 MB） | 可編輯 Q&A 清單 | 手冊、規格、法規等知識點萃取 |
| **筆記 → FAQ** | 貼上文字 | 可編輯 Q&A 清單 | 會議紀錄、草稿、口述重點 |
| **SOP → FAQ** | PDF、TXT | 可編輯 Q&A 清單 | 操作程序；**不合併、不省略步驟** |
| **文件 → 結構化 MD** | **PDF only** | 可編輯 Markdown | 章節化手冊、含圖／掃描頁的 PDF |
| **網頁 → MD** | 公開 **URL** | 可編輯 Markdown | 產品說明頁、Help 文章、政府公開資料 |
| **批次 → 結構化 MD** | 多個 PDF | 本機 `.md` 檔 | 大量離線轉檔，再自行上傳 KM 或後續處理 |

> **文件 → FAQ** 與 **SOP → FAQ**：前者著重萃取知識點，可整合相近內容；後者嚴格保留每個步驟。

> **文件 → 結構化 MD** 僅支援 PDF。Word 請先匯出 PDF；一般網頁可改用 **網頁 → MD**。

---

## 共用：處理進度視窗

開始整理或批次轉換時，畫面中央會顯示 **處理中** 對話框：

- 顯示目前步驟（例如 OCR 頁數、AI 結構化第 N/M 段、批次第幾檔）
- 長文件可能需 **數分鐘**，請勿關閉分頁
- 可點 **取消** 中止（已產出的 Q&A 或 MD 片段會保留）

背景仍會逐段顯示結果（Q&A 卡片或 Markdown 文字）。

## FAQ 模式（文件 / 筆記 / SOP）

### 第一步：輸入內容

**文件 → FAQ / SOP → FAQ**

| 項目 | 說明 |
|------|------|
| **支援格式** | PDF、TXT（最大 20 MB） |
| **上傳方式** | 拖曳至上傳區，或點擊選擇檔案 |
| **開始** | 點「開始智慧整理」（文件）或上傳後於編輯畫面操作（SOP 直接進編輯畫面） |

**筆記 → FAQ**

- 左側貼入或輸入文字
- 點「轉成 Q&A」開始整理
- 每次轉換 **預設累加** 至既有 Q&A 清單

> 掃描式 PDF（純圖片、無文字層）在 FAQ 模式無法擷取文字，請改用含文字層的 PDF，或先使用 **OCR Agent**。  
> 若需保留圖中組織圖、表格等，請改用 **文件 → 結構化 MD**（內建 OCR）。

---

### 第二步：審閱與編輯

- **文件 / SOP 模式**：左側原始文件預覽，右側 Q&A 卡片
- 每一條 Q&A 可直接編輯、刪除；底部可「新增一條」；標題列可「清空」全部

| 操作 | 說明 |
|------|------|
| **編輯** | 直接修改問題／答案欄位 |
| **刪除** | 卡片右上角刪除 |
| **新增** | 底部「＋ 新增一條」 |
| **清空** | 標題列「清空」（需確認） |

---

### 第三步：AI 改寫（選用）

針對 **單條** Q&A 微調，無需重跑整份文件：

1. 點卡片右上角紫色改寫圖示（✦）
2. 選擇預設指令（可多選）或輸入自訂指令
3. 點「改寫」→ 確認後「套用」或「捨棄」

---

### 第四步：下載與匯入知識庫

點「**下載 Q&A**」：

| 欄位 | 說明 |
|------|------|
| **Q&A 集名稱** | 下載 TXT 檔名與匯入來源名稱 |
| **同時匯入至知識庫** | 可選現有知識庫或建立新知識庫 |

確認後會下載 TXT 備份；若開啟匯入，Q&A 以 **FAQ 類型** 寫入知識庫（利於問答檢索）。

**跨文件累加**：文件 / SOP 重新上傳時可選「累加模式」，新 Q&A 附加在既有清單後。

## 網頁 → 結構化 MD

將公開網頁轉成帶章節的 Markdown，流程：**URL → 正文預覽 → 確認 → PDF 快照 → 結構化**（與 PDF 管線相同）。

### 流程

1. 貼上 **http(s) URL**，點「擷取正文預覽」
2. 系統自動去除導覽列、頁尾等常見雜訊，顯示正文預覽
3. 可調整標題，確認無誤後點「**確認並結構化**」
4. 完成後可下載 `.md`、匯入知識庫（`structured_md`）

### 限制（Phase 1）

- 僅 **可公開存取** 的網頁；登入牆、重度 JavaScript 動態站可能失敗
- 後端以 **trafilatura** 抽取正文；後續可再強化（例如 Playwright）
- 失敗時請改 **文件 → 結構化 MD** 上傳 PDF

匯入知識庫時 YAML 會記錄 `source_url` 供追溯。

## 文件 → 結構化 MD

將 PDF 轉成帶 `##` / `###` 章節標題的 Markdown，並保留表格、條列與 OCR 區塊的層級結構。

### 流程

1. 上傳 **PDF**（拖曳或點選）
2. 點「**開始結構化**」
3. 左側 PDF 預覽，右側 Markdown 編輯區（處理中內容逐段出現）
4. 完成後可 **複製**、**下載 .md**、**匯入知識庫**

### 自動 OCR

管線會在下列情況呼叫 Vision OCR（與 OCR Agent 共用影像辨識能力）：

- PDF **低文字頁**（例如掃描頁、心智圖頁）
- 頁面內 **內嵌大圖**（組織圖、流程圖等）

OCR 結果以 `[OCR 補充]`、`[圖片 OCR · …]` 等標記嵌入 MD，再交由 AI 補上章節標題。**含圖 PDF 處理時間較長**，請留意進度視窗。

### 匯入知識庫

點「**匯入知識庫**」→ 選擇或新建知識庫。系統以 **`structured_md`** 類型寫入，並依 Markdown 標題切分 chunk（含章節麵包屑），適合 Bot **AI 生成回答** 模式下的長文件檢索。

匯入前建議人工快速檢查標題與表格是否正確。

## 批次 → 結構化 MD

一次轉換多份 PDF，輸出至本機資料夾（或逐檔下載）。

### 操作步驟

1. **加入檔案**：選擇一或多個 PDF
2. **選擇輸出資料夾**（Chrome / Edge 支援；其他瀏覽器改為每檔完成後個別下載）
3. 點「**開始轉換**」
4. 列表顯示每檔狀態：等待 / 處理中 / 已存檔 / 失敗

### 行為說明

| 項目 | 說明 |
|------|------|
| **只跑待處理** | 開始時僅處理「等待」與「失敗」項目；**已存檔** 不會重跑 |
| **新一輪** | 上一輪全部完成後，再「加入檔案」會 **取代** 列表，開始新一輪 |
| **停止** | 可隨時中止；已完成檔案保留在輸出資料夾 |
| **清空列表** | 非執行中時可清空 |

批次模式 **不直接匯入知識庫**；產出的 `.md` 可於 Knowledge Base 上傳（含 Doc Refiner YAML 標記時會自動識別為 structured_md），或回到單檔模式編輯後匯入。

---

## 重要特性

- **六種模式獨立狀態**：各模式資料互不干擾
- **瀏覽器暫存**：Q&A、筆記文字、MD 內容與標題自動存於本機，重新整理後仍保留（處理中暫不寫入 MD）
- **分段串流**：長文件自動切段；FAQ 與 MD 皆逐段顯示，無需等全部完成
- **Sidebar 模型**：所有模式共用同一選定 LLM

---

## 注意事項

- FAQ 模式：**PDF、TXT**；結構化 MD：**僅 PDF**。
- 純掃描 PDF 在 FAQ 模式無法使用；結構化 MD 可透過內建 OCR 處理多數情況，品質仍建議人工抽查。
- AI 輸出僅供參考，**匯入知識庫前請審閱**（必要時使用 AI 改寫或手動編輯 MD）。
- 含大量 OCR 的 PDF 可能處理 **十數分鐘**，請保持分頁開啟並留意進度視窗。
- 長文件（逾約 20,000 字）FAQ 模式會自動分段整理。
- 公司共用知識庫匯入需 **管理員 / 主管** 權限；個人知識庫僅建立者可匯入（管理員除外）。
