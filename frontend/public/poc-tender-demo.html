LLM 角色詳述（Steps 03–05）

這個 POC 與停車報價 POC 最大的不同：LLM 不只是「生成文字」， 更要做「文件理解」和「推論比對」，是整個產品的核心引擎。

資料來源：g0v 標案 API

可立即串接
不需要爬蟲，有現成合法 API
g0v 社群維護的 pcc.mlwmlw.org/api 直接提供政府採購網結構化資料，開放 CORS、免費使用， 資料來源符合著作權法合理使用規定（報導、研究、正當目的）。
用途Endpoint說明
關鍵字搜尋 /api/keyword/電子收費 最常用，直接對應 Step 01 的關鍵字設定
每日新增標案 /api/date/tender/2026-05-31 排程每日執行，取得當天所有公告標案
每日得標公告 /api/date/award/2026-05-31 分析競爭者、了解市場行情
機關月別查詢 /api/unit/交通部/2026-05 鎖定特定主管機關，追蹤相關標案
使用注意事項
g0v API 因大量 AI 爬蟲造成負載，已加 Cloudflare 防護並阻擋機器人行為。 請設定合理 rate limit（建議每分鐘不超過 10 次請求）並加上正常的 User-Agent。 商業上線前建議法務確認政府資料商業使用的合規性。

關鍵差異
停車 POC 的核心是「財務計算引擎」，LLM 是輔助； 標案 POC 的核心是「LLM 文件理解」，沒有 LLM 就沒有產品。 兩個 POC 互補，展示 AI 在不同場景的不同角色。
