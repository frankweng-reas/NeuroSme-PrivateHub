# 商業洞察助理

## 快速上手：建立資料範本 → 建立主題 → 選擇資料範本後匯入資料 → 開始對話

---

## ✦ 什麼是商業洞察助理？

商業洞察助理採用 **Agentic AI** 架構，能主動規劃並執行多步驟分析流程，回答複雜的商業問題。

你只需要用自然語言提問，助理會自動：

1. **理解問題**，拆解成若干個查詢任務
2. **逐步查詢**，依照前一個結果決定下一步怎麼查
3. **整合結果**，統整成一份完整的 Markdown 分析報告

這不是簡單的關鍵字搜尋，而是真正理解業務脈絡的自動化分析流程。

---

## ✦ 適用各種業務場景

商業洞察助理並不限定於特定產業或分析類型。只要上傳一份 CSV，定義好欄位範本，就能立即進行智慧分析：

| 場景 | 資料示例 |
|---|---|
| 銷售分析 | 訂單明細、通路業績、品牌表現 |
| 財務分析 | 損益表、費用明細、現金流量 |
| 人資管理 | 員工考核、薪資結構、出缺勤 |
| 庫存管理 | 物料庫存、安全庫存缺口、補貨建議 |
| 客戶分析 | 消費行為、RFM 分層、客戶終身價值 |
| 行銷成效 | 廣告投放、點擊轉換、渠道 ROI |
| 生產製造 | 產線良率、不良原因、設備稼動率 |

**換一份資料範本，就換一個分析領域，不需要改任何設定。**

## ✦ Agentic 多步驟分析

傳統 BI 工具需要你手動設計查詢、選擇維度、執行報表。商業洞察助理可以**自主規劃整個分析流程**，例如：

> **「分析今年各季的銷售表現，找出最佳與最差的通路，並說明可能原因」**

助理會自動依序執行：
- 第 1 步：查詢各季各通路銷售總額
- 第 2 步：計算季度環比成長率
- 第 3 步：識別最佳／最差通路
- 第 4 步：整合以上資料，撰寫分析報告

這些步驟**全部自動完成**，不需要你一問一答。

### 適合問的問題類型

| 問題類型 | 範例 |
|---|---|
| 趨勢分析 | 過去三個月的銷售走勢？哪個月份異常？ |
| 比較分析 | A、B、C 三個通路的績效比較？ |
| 根因分析 | 上個月業績下滑的可能原因？ |
| 明細查詢 | 庫存不足的物料清單？缺口最大的前 20 項？ |
| 預測輔助 | 依歷史趨勢預估下週來客數？ |
| 複合查詢 | 先找出銷售前 10 的客戶，再分析他們的購買週期 |

## ✦ 多主題跨平台分析

當你同時管理多個通路或業務單位（例如：momo、PChome、91app 各一份資料），可以一次選取多個主題，讓助理同時查詢各主題並進行橫向比較。

> **「各平台 2026 年第一季的銷售狀況？」**

助理會自動：
1. 對每個選取的主題分別查詢
2. 彙整各平台的結果
3. 整合成一份跨平台比較報告

### 使用方式（桌機版）

目前桌機版每次分析對應**單一主題**。若需要多主題跨平台分析，請使用**手機版介面**（見下方說明）。

### 主題共用

分析主題的**建立者**可以將主題共用給同組織的其他使用者：

1. 點選主題旁的「⋯」選單 → **修改專案**
2. 在「**共用給**」區塊勾選要授權的使用者
3. 儲存後，對方登入即可看到並使用此主題（僅限查詢，無法修改或刪除）

## ✦ 手機版介面（多主題分析）

點擊右上角**手機圖示**，可取得手機版入口連結或掃描 QR Code，在手機或平板開啟專屬的對話介面。

手機版支援：
- **多主題同時選取**：可選取多個分析主題，一次詢問跨主題問題
- **對話紀錄保留**：切換分頁或重新整理後，對話進度不會遺失
- **選取主題記憶**：上次選擇的主題在重新整理後自動還原

### 操作說明

| 功能 | 操作 |
|---|---|
| 選擇主題 | 點擊頂部主題名稱按鈕 |
| 多選主題 | 在主題選擇畫面勾選多個主題 |
| 查看分析步驟 | 分析進行中，底部自動彈出步驟進度 |
| 範例問題 | 點擊輸入框旁的燈泡圖示，多主題時依主題分組顯示 |
| 加入主畫面 | 使用瀏覽器「加入主畫面」功能，手機版可當作 App 使用 |

> **多主題分析提示**：選取 2 個以上主題時，建議在問題中明確說明想比較的面向（例如：「比較各平台的客單價與退貨率」），有助於 AI 產出更精確的跨主題報告。

## ✦ 分析結果的使用

### 下載 CSV
分析報告中的每一個表格，下方都有「**下載 CSV**」按鈕，可直接匯出為 Excel 可讀取的格式（含 UTF-8 BOM）。

### 圖表視覺化
含數值欄位的表格，下方會出現「**圖表**」按鈕，支援：
- **長條圖**（預設）：適合比較各類別
- **折線圖**：適合時間序列趨勢（系統自動偵測時間格式）
- **圓餅圖**：適合佔比分析

## ✦ 操作說明

### 分析主題（左側）

- **選擇或建立主題**：每個主題對應一組分析資料（與資料模型綁定）。
- **主題旁的資料庫圖示**：
  - **綠色**：已匯入資料，可進行分析
  - **紅色**：尚無資料，需先匯入
  - **琥珀色**：確認資料狀態中

### 匯入資料

1. 點主題旁的資料庫圖示，或選單中的「匯入資料」
2. 選擇**資料範本**
3. 上傳符合範本格式的 **CSV 檔案**
4. 匯入完成後即可開始提問

> 資料模型範本的新增與管理，請點右上角「**資料模型範本**」按鈕，並參考該頁面的說明。

> **多張表格的資料**：目前每個分析主題對應一張資料表。若資料分散於多個表格（例如銷售明細 + 商品主檔），請先在 Excel 或其他工具中以 VLOOKUP / 合併查詢等方式整合成一張寬表，再匯入 CSV。

### 對話區功能

| 功能 | 說明 |
|---|---|
| 範例問題 | 管理常用問題，一鍵帶入輸入框 |
| 資料庫圖示 | 顯示目前資料筆數，點擊進行資料管理 |
| 清除對話 | 清空目前對話記錄 |

## ✦ 進階設定：提供業務背景資訊

右側「**進階設定**」中可輸入自訂提示詞（User Prompt），讓助理在分析時參考你提供的業務背景，進行更深入的交叉分析。

### 典型應用：資料 × 背景知識交叉分析

當你有一份銷售資料，同時想對照促銷活動、節慶或其他外部事件進行分析，**資料庫不需要額外的促銷資料**，只需在進階設定中說明：

```
促銷活動背景：
- 3/8 婦女節：Momo 通路燕麥奶品類買一送一
- 3/15 白色情人節：全通路甜食類 85 折
- 3/25 週年慶：PChome 全館 9 折起
```

提問時只需說：「分析 3 月銷售成長與促銷活動的關聯」，助理會自動對照你提供的背景，說明哪段銷售高峰與哪項活動時間吻合。

### 其他適合放在進階設定的背景資訊

- 產品分類定義（例如：A 系列為高毛利商品）
- 季節性說明（例如：每年 Q1 為淡季）
- 通路特性描述（例如：Momo 以量販為主，客單價較低）
- 分析重點方向（例如：請特別關注毛利率而非營收）

> 助理會忠實引用你提供的背景資訊，並在報告中標示「依據您提供的背景資訊」，不會自行補充你未提及的內容。

## ✦ 注意事項

- **資料留在本機**：所有資料儲存於您的環境，不會上傳至雲端。
- **每輪分析獨立**：每次提問都是全新分析，助理不會記憶上一輪的對話內容。若要延續分析，請在問題中包含完整的背景說明。
- **資料量限制**：若查詢結果非常大，AI 分析將以完整資料進行，若超出模型上限將顯示提示，建議縮小時間範圍或加入篩選條件後重試。
- **AI 設定區（右側）**：可調整提示詞、分析角色等，請查看右側 Help 說明。
