# KB Bot Builder 使用說明

## Overview

**KB Bot Builder** 以企業知識庫為基礎，建立 AI 問答 Bot，並部署至所需場景。

| 部署情境 | 說明 |
|---------|------|
| **對外客服 Widget** | 嵌入官網、產品頁面，讓訪客自助查詢產品資訊、退換貨政策等 |
| **內部員工 Bot** | 嵌入企業內部系統（OA、HR 入口），讓員工查詢 HR 政策、IT 規範、操作手冊 |
| **API 串接** | 透過 API Key 讓現有系統直接呼叫 Bot，不需要嵌入 Widget |

知識管理群組的三個 Agent 分工如下：
- **Doc Refiner** → 整理 PDF 為 Q&A 後匯入知識庫
- **Knowledge Base** → 管理文件、直接查詢
- **KB Bot Builder**（本 Agent）→ 以知識庫為基礎部署 Bot

---

## 功能定位

KB Bot Builder 讓使用者以公司知識庫為基礎建立 AI Bot，並部署為嵌入式 Widget 或 API，供內部系統或外部網站使用。
建立 Bot 前，請先在 **Knowledge Base** 建立並設為「公司共用」的知識庫。

---

## 1. Bot 列表（左側欄）

左側欄列出本租戶下所有 Bot。

| 操作 | 說明 |
|------|------|
| **新增 Bot** | 點擊上方「＋」，輸入名稱後建立 |
| **切換 Bot** | 點選任一 Bot，右側立即切換至該 Bot 的設定與測試 |
| **設定** | 點「⋯ → 設定」進入 Bot 詳細設定 |
| **刪除** | 點「⋯ → 刪除」，**此操作無法復原** |

---

## 2. Bot 設定

| 設定項目 | 說明 |
|---------|------|
| **Bot 名稱** | 顯示名稱，也用於識別 Bot |
| **存取控制** | 決定 Widget 的使用對象（詳見下方說明） |
| **AI 模型** | 指定此 Bot 使用的 LLM，留空採系統預設 |
| **回答模式** | 決定 Bot 如何回覆問題（詳見下方說明） |
| **知識來源（KB）** | 依回答模式過濾，勾選此 Bot 引用的公司知識庫（可多選） |
| **Bot 角色與語氣設定** | 設定 Bot 的角色、語氣與業務規則 |
| **找不到答案時的回覆** | 啟用後，Bot 無法從知識庫找到答案時，回傳自訂訊息而非 AI 猜測 |
| **客服情境功能** | 首頁面、熱門 FAQ、常見 FAQ、聯絡資訊（詳見下方說明） |

> 知識來源只顯示「公司共用」知識庫，個人知識庫不適用於對外 Bot。


### 存取控制

決定誰可以開啟使用此 Bot 的 Widget：

| 模式 | 說明 | 適用場景 |
|------|------|---------|
| 🔓 **公開模式** | 任何人可直接開啟 Widget，無需登入 | 對外客服、官網 FAQ、公開問答 |
| 🔒 **內部認證模式** | 使用者須以公司 LocalAuth 帳號登入後才能使用 | HR 規章、內部 SOP、限制同仁存取 |

> **與 KB 範圍的關係**：KB 範圍控制「誰能在後台管理知識庫」；Bot 存取控制決定「誰能使用前端 Widget」，兩者獨立設定，可靈活組合：
> - 公司 KB + 公開 Bot → 對外服務
> - 公司 KB + 認證 Bot → 內部專屬

### 回答模式

| 模式 | 說明 | 適用情境 |
|------|------|---------|
| **AI生成回答** | AI 依知識庫內容生成自然語言回覆 | 一般問答、文件查詢、客服 |
| **精準回答** | 直接回傳 FAQ 原文（Q&A），不經 AI 改寫 | 法規條文、人事規章、標準 SOP |

- 選擇回答模式後，知識來源清單會自動過濾，只顯示符合該模式的知識庫。
- **切換回答模式時，已選擇的知識庫會自動清空**，請重新選擇。
- 知識庫的回答模式在「Knowledge Base」中設定。

### Bot 角色與語氣設定

- 設定 Bot 的角色名稱、語氣與業務限制（例如：只回答產品相關問題）。
- **核心行為約束**（只根據知識庫回答、找不到回覆 NOT_FOUND、拒絕探詢系統設定）由系統自動套用，無需填寫。
- 透過 Bot 引用知識庫時，Bot 的設定完全覆蓋知識庫本身的模型與提示詞設定。

## 3. 部署

### 服務狀態

| 狀態 | 說明 |
|------|------|
| **運行中** | Bot 可正常服務；Widget 與 API 均可使用 |
| **已暫停** | Bot 暫停服務；Widget 與 API 均無法使用 |

### 嵌入式 Widget

Widget Token 由 **系統管理員** 在「管理後台 → Bot 部署管理」統一開通與管理。

- 若 Token 已開通，此頁面將顯示 Widget URL 與嵌入碼（`<script>` 標籤）。
- 將嵌入碼貼入目標網頁 `<body>` 底部，Widget 即會在頁面右下角顯示聊天按鈕。
- 若尚未開通，請聯繫系統管理員申請部署。

> Widget Token 的產生與撤銷僅限 admin 操作，請至「管理後台 → Bot 部署管理」進行。

## 4. 測試 Chat

選取 Bot 後可直接進行對話測試。

- 輸入問題，測試 Bot 是否能根據知識庫正確回答。
- AI 回答附帶**來源標注**（來源：檔名），確認答案是否來自正確文件。
- 對話記錄會保留（local storage），離開頁面再回來仍可查看。
- 點擊右上角重新整理圖示可清除本段對話。
- 輸入框支援 **Shift + Enter** 換行，**Enter** 送出訊息。

---

## 5. 客服情境功能

針對客服應用場景，可在 Bot 設定中啟用以下功能，讓 Widget 提供更完整的服務體驗：

### 首頁面

啟用後，新訪客開啟 Widget 時會先看到首頁面，再進入對話。

| 設定 | 說明 |
|------|------|
| **啟用/關閉** | 開關切換 |
| **歡迎語** | 首頁面頂部顯示的歡迎訊息 |

### 熱門 FAQ

在 Widget 首頁面以卡片方式展示最常見的問答，訪客可直接點開查看答案。

- 可個別新增、編輯、刪除、排序
- 答案支援 Markdown 格式（連結、粗體、換行等）
- 啟用需同時開啟「首頁面」

### 常見 FAQ

透過 Widget 右上角 ❓ 圖示開啟的常見問題列表，以卡片方式展示，適合放置較多問題供訪客自助查閱。

- 可個別新增、編輯、刪除、排序
- 與「熱門 FAQ」獨立管理，可分別開關

### 聯絡資訊

透過 Widget 右上角 📞 圖示開啟的聯絡方式清單，讓訪客可直接撥打電話、發送 Email 或開啟外部連結。

| 聯絡類型 | 說明 |
|---------|------|
| 📞 電話 | 點擊自動撥號 |
| 📧 Email | 點擊開啟郵件客戶端 |
| 💬 Line | 開啟 Line 連結 |
| 📋 表單 | 連結至 Google 表單等外部表單 |
| 🌐 網址 | 任意外部網址 |

## 6. 查詢統計

顯示此 Bot 的查詢使用狀況，協助了解 Bot 的服務效益。

| 功能 | 說明 |
|------|------|
| **時間範圍** | 可選近 7 天、30 天、90 天 |
| **最多人問** | 列出查詢頻率最高的問題，可作為知識庫優化依據 |
| **零命中** | 列出 Bot 無法回答的問題（知識庫中找不到對應資料），優先補充這些內容可顯著提升回答率 |

---

## 7. 訪客對話

記錄外部用戶透過 Widget 與 Bot 的對話紀錄。

- 選取 Session 可查看完整對話內容。
- 點擊「**複製對話**」可將整段對話以純文字格式複製至剪貼簿，方便貼入其他 Agent（如 Chat Agent）進行進一步分析。
- 可用於查核 Bot 回答品質、了解使用者常見問題。

---

## 8. API 整合

提供 API Key 讓外部系統直接呼叫 Bot 問答功能（不透過 Widget）。

| 操作 | 說明 |
|------|------|
| **產生 API Key** | 每個 Key 格式為 `nsk_...` |
| **撤銷 API Key** | 撤銷後該 Key 立即失效 |
| **查看使用量** | 顯示各 Key 的請求次數與 token 用量 |

**呼叫端點：**

```
POST /api/v1/public/bot/query
X-API-Key: nsk_your_key_here
Content-Type: application/json

{
  "bot_id": 1,
  "question": "請問退貨政策是什麼？"
}
```

Rate limit：每個 API Key 每小時最多 100 次。詳細規格請參閱 `/api/v1/public/docs`。

## 10. 注意事項

- 刪除 Bot 將同時刪除其所有訪客對話紀錄與 API 使用紀錄，**無法復原**。
- Widget Token 撤銷後，已嵌入的 Widget 將**立即停止運作**，對話紀錄不受影響。
- Bot 啟用/停用會同時影響 Widget 與 API，請確認後再操作。
- 公司知識庫的文件如有異動（新增/刪除），Bot 下次查詢即時反映，無需重新部署。
